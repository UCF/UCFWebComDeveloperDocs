
This vault is a living reference for the UCF Web Communications development team. The goal is to document the systems, processes, and conventions we work with day-to-day — so whether you're onboarding, picking up an unfamiliar project, or just trying to remember how something works, you should be able to find a straight answer here.

Think of it less as a formal wiki and more as a shared notebook that happens to be well-organized.

## A Few Ground Rules

- **Keep it current.** Outdated docs are worse than no docs. If something changes, update the relevant page.
- **Link generously.** If a page references something documented elsewhere in the vault, link to it.
- **Document the why, not just the what.** Future-you will thank present-you.
